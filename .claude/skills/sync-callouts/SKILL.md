# Sync Callouts

Sync custom admonition and callout styles from the Obsidian vault at `~/Desktop/digital-brain/` into `src/site/styles/user/callouts.scss`.

## Sources

1. **Admonition plugin config**: `~/Desktop/digital-brain/.obsidian/plugins/obsidian-admonition/data.json` тАФ read `userAdmonitions` for custom type definitions (name, color, icon)
2. **CSS snippets**: `~/Desktop/digital-brain/.obsidian/snippets/*.css` тАФ scan for `.callout[data-callout="..."]` rules with `--callout-color` and `--callout-icon` overrides. Snippet values take precedence over plugin values.

## Process

1. Read the admonition plugin `data.json` and extract each entry from `userAdmonitions`:
   - `color` тАФ RGB string like `"116, 227, 161"` or hex like `"#7d7d7d"` (convert hex to RGB)
   - `icon.name` + `icon.type` тАФ resolve to a Lucide icon name (see mapping below)
2. Read all `.css` files in the snippets directory. For each `.callout[data-callout="X"] { ... }` rule, extract `--callout-color` and `--callout-icon` values. These override the plugin values.
3. Skip standard Obsidian types (note, abstract, summary, tldr, info, todo, tip, hint, important, success, check, done, question, help, faq, warning, caution, attention, failure, fail, missing, danger, error, bug, example, quote, cite) тАФ only sync custom types.
4. Generate `src/site/styles/user/callouts.scss` with the merged results.

## Icon Resolution

The blog uses Lucide icons loaded from CDN. The existing `calloutScript.njk` reads `--callout-icon` from CSS and renders it client-side.

- **Obsidian/Lucide icons** (`icon.type === "obsidian"` or `"lucide"`): use the name directly. Add `lucide-` prefix if not present.
- **Font Awesome icons** (`icon.type === "font-awesome"`): map to Lucide equivalents:

If a Font Awesome icon has no direct mapping:
1. Search the Lucide icon set for a close match and suggest it to the user
2. If the user doesn't like the suggestion, ask them to provide a Lucide icon name
3. If the user provides nothing or wants to skip, fall back to the default pencil icon

All icon values in the SCSS output must use the `lucide-` prefix (e.g., `lucide-bot`).

## Output Format

Write **only** the type definitions to `src/site/styles/user/callouts.scss`. Structural overrides (icon sizing, nested callout margins, etc.) live separately in `src/site/styles/user/callout-overrides.scss` and must NOT be duplicated here.

```scss
// Auto-generated by /sync-callouts тАФ do not edit manually.
// Source: Obsidian vault admonition plugin config + CSS snippets.
// Run /sync-callouts to regenerate.

.callout[data-callout="type-name"] {
  --callout-color: R, G, B;
  --callout-icon: lucide-icon-name;
}

// ... one block per custom type
```
